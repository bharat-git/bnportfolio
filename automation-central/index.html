Automation Central

The challenge

Automation Central, a multipurpose repository for reusable components, tools and case studies which are stored as assets. Lately, this portal was not being used widely among the employees due to lacklustre UI and few user known pain points. To overcome this, Infosys organized a prototyping contest to bring out the best design possible and revamp automation central.

The original web site looked like this .

Pain Points

  •  The UI is not aesthetically appealing
  • In the Home page(image 1) there is no option to search for the assets. One has to go into the categories like tools, platforms and components to view the whole list of assets available.
  • The individual asset page (image 3) irrelevant information like assets downloads by month etc. were given. It is already understood from the downloads section next to the asset name and version.

Improvements

  • Search bar on the top of the home page and category wise filters to sort relevant assets.
  •  Cool and new slide down details section on asset cards to save space for relevant information.
  •  Improved discussions section UI and a chat-it-out (embedded chat-bot) section for queries and other suggestions. 
  •  Improved assets page with all asset details, artifacts, discussions and reviews section which users can view in one glance. 

The Final result

The main impediment for me was to create the working prototype in iRise (a prototyping tool) as per the rules of the competition. I had previously worked on creating prototypes using Adobe XD but creating a working prototype in an altogether new tool was a challenge in itself. 

I have captured some screenshots of the final working prototype so you can get an idea.
